HEVC with alpha基本原理理：

  • ⼀个视频⽂件中有若干视频轨

  • 一个视频轨有若干视频帧

  • HEVC video就是其中⼀个视频轨

  • 每个视频帧有两个layer，Base Layer & Alpha Layer,每个layer都是用HEVC

  • codec编码压缩

    • Base Layer是纯⾊色层
    • Alpha Layer是透明层
    • 两个层的定义是遵循HEVC公开标准的，意味着没法处理理alpha layer的HEVC解码器会忽略略alpha layer，只解码base layer

相关的API使⽤用：

生产端

AVAssetWriter：

你可以在应⽤用⾥里里渲染Apple ProRes4444视频，⽤用AVAssetWriter把视频转成HEVC with alpha，

AVAssetExportSession：

AVAssetExportSession 和他的新preset
AVAssetExportPresetHEVC…WithAlpha将任何⽀支持alpha通道的视频转成HEVC with Alpha
AVAssetExportSession 将透明背景的HEVC with Alpha视频转成带纯⾊色背景的普通视频，⽐如不支持Alpha通道的普通H.264视频，以便在不支持HEVC的设备播放
VTCompressionSession可以直接⽤用VideoToolBox将采集的视频帧编码成HEVC with alpha

消费端:

AVPlayer通过 AVPlayerLayer渲染HEVC with Alpha
AVPlayerItemVideoOutput 访问透明视频帧
AVAssetImageGenerator从HEVC with Alpha 提取⼀一个透明背景的CGImage
AVAssetReader读取HEVC with Alpha的视频帧
VTDecompressionSession解码HEVC with Alpha 的sample buffers

透明度⽀支持两种，⼀一种是alpha作⽤用到RGB，⼀一种是纯⾊色透明分开，
Premultiplied Alpha是推荐的，Session中没解释两种区别，猜测是Premultiplied
Alpha不不会把半透明的地⽅方处理理成纯⾊色，⽽而是保留留原视频帧透明度

设置outputSettings和CVBufferSetAttachment时候⼀一定设置成匹配的，否则会
